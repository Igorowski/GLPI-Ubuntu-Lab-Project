# Software Installation Request

## Overview
This project simulates a standard IT service request handled in a GLPI environment.  
The scenario covers a software installation request for Microsoft Teams in a Windows domain environment and demonstrates the full ticket lifecycle from user request to deployment and verification.

---

## Scenario
A user requests installation of Microsoft Teams on their workstation for business communication and collaboration purposes.

---

## User Request
- I need a software installed on my workstation for work purposes. Please install Microsoft Teams and ensure it is properly configured.

## Environment
- Windows 10 client workstation  
- Active Directory domain environment  
- Software deployment via IT support (manual installation in lab scenario)  
- GLPI ITSM ticketing system  

---

## Technician Actions
1. Verified software request and availability  
2. Checked installation permissions and compatibility  
3. Downloaded and prepared Microsoft Teams installer  
4. Installed application on target workstation  
5. Performed initial launch and functionality check  
6. Confirmed successful installation and readiness for use
